# Parrot V1 翻译偏好 PRD

版本：0.2 Draft  
日期：2026-06-23  
平台：macOS  
适用范围：MVP 后续 V1 功能规划  
前置文档：`Docs/ai-translation-macos-prd.md`

## 1. 背景

Parrot 的 MVP 已完成菜单栏常驻、快捷文本翻译、截图 OCR 翻译、原文译文对照、Provider 设置、Keychain 密钥保存、翻译历史和自定义快捷键。

当前翻译链路仍以自动中英互译为主，适合最短路径使用，但还不能覆盖更细的工作场景：

- 用户需要固定翻译方向，例如中文写作时长期翻译成英文。
- 用户需要临时切换语言，例如英文、日文、韩文、法文、西班牙文之间切换。
- 用户希望看到默认 Prompt，并按自己的写作或工作习惯微调。
- 用户希望固定术语翻译，例如产品名、专有名词、团队内部词汇。
- 用户希望不用点菜单，也能通过快捷键直接打开设置页。

V1 的目标不是把 Parrot 做成完整翻译工作台，而是在保持轻量的前提下，让语言、Prompt 和术语真正进入日常翻译链路。

## 2. 产品目标

### 2.1 V1 目标

- 支持通过全局快捷键打开 Settings。
- 在快速翻译和截图 OCR 翻译界面展示源语言和目标语言。
- 支持源语言和目标语言一键互换，并可立即重新翻译。
- 在 Settings 中展示默认翻译 Prompt，并支持启用自定义 Prompt。
- 支持个人本地术语表，翻译时优先遵循命中的术语。
- 为常见翻译风格提供低门槛配置，降低直接编辑 Prompt 的成本。

### 2.2 成功标准

- 日常翻译仍不超过 2 次显式操作。
- 用户可以在翻译窗口内完成语言切换和互换，不必先进入 Settings。
- 用户可以恢复默认 Prompt，避免自定义配置损坏后无法回退。
- 术语表只在源文本命中时注入翻译请求，不明显增加短文本翻译延迟。
- 所有新配置不保存 API Key，不保存截图图片，不改变默认隐私边界。

## 3. 非目标

V1 不做以下能力：

- 团队术语库、云同步术语库或权限管理。
- 专业 CAT 工具能力，例如翻译记忆库、项目管理、句段确认。
- 大型术语文件导入导出，例如 TBX、TMX。
- 图片直接上传给多模态模型。
- 多 Prompt profile 市场或 Prompt 模板商店。
- 跨平台适配。

## 4. 信息架构调整

Settings 从当前的 `Model`、`Shortcuts`、`Privacy` 扩展为：

- `Model`：Provider、Base URL、模型、API Key、连接测试。
- `Shortcuts`：快速文本翻译、截图翻译、打开设置。
- `Translation`：默认语言、翻译风格、Prompt、术语表。
- `Privacy`：翻译历史、本地数据说明、清空数据入口。

`Translation` 是 V1 的核心设置区，不提前加入无真实行为支撑的空壳项。

## 5. 功能需求

### 5.1 设置快捷键

用户可以为 `Open Settings` 配置全局快捷键。

默认建议：

- `Quick Text Translation`：`Cmd+Shift+T`
- `Screenshot Translation`：`Cmd+Shift+2`
- `Open Settings`：`Cmd+Option+,`

要求：

- 与现有快捷键录制能力一致，支持录制、保存、恢复默认和冲突提示。
- 保存后立即重新注册全局快捷键，不要求重启 App。
- 当快捷键被系统或其他 App 占用时，Settings 中展示可理解的错误。
- 菜单栏 `Settings` 入口继续保留。

### 5.2 语言选择与一键互换

快速翻译窗口和截图翻译结果窗口都展示轻量语言栏。

语言栏包含：

- 源语言选择：`自动`、`中文`、`英文`、`日文`、`韩文`、`法文`、`西班牙文`。
- 目标语言选择：`自动相反语言`、`中文`、`英文`、`日文`、`韩文`、`法文`、`西班牙文`。
- 一键互换按钮。
- 重新翻译按钮。

默认行为：

- 源语言为 `自动`。
- 目标语言为 `自动相反语言`。
- 中文输入默认译为英文。
- 英文输入默认译为简体中文。
- 混合语言输入默认译为用户设置的默认目标语言。

互换规则：

- 如果源语言和目标语言都是明确语言，点击后直接互换。
- 如果源语言是 `自动`，目标语言是明确语言，点击后源语言变为原目标语言，目标语言变为 `自动相反语言`。
- 如果目标语言是 `自动相反语言`，点击后优先基于最近一次检测结果互换。
- 源语言和目标语言不能保存为同一种明确语言，出现冲突时禁用翻译并提示。

### 5.3 翻译风格

用户可以选择翻译风格，风格影响 Prompt，但不改变 Provider 配置。

首批风格：

- `准确`：忠实原意，适合资料阅读和报错理解。
- `自然`：表达更顺，适合聊天和日常写作。
- `专业`：保持正式和术语一致，适合文档、邮件和产品材料。
- `简洁`：压缩表达，适合快速理解和摘要式转述。

默认风格为 `准确`。

### 5.4 默认 Prompt 展示与自定义 Prompt

Settings 的 `Translation` 区展示当前默认 Prompt。

用户可以启用自定义 Prompt 模板。模板支持以下变量：

- `{source_language}`：源语言。
- `{target_language}`：目标语言。
- `{style}`：翻译风格。
- `{glossary}`：命中的术语列表。
- `{text}`：待翻译文本。

要求：

- 默认 Prompt 始终可见。
- 用户可以启用或关闭自定义 Prompt。
- 用户可以恢复默认 Prompt。
- 保存前校验必须包含 `{target_language}` 和 `{text}`。
- 如果模板无效，翻译链路继续使用默认 Prompt，并在 Settings 中提示错误。
- Prompt 不允许记录 API Key、Provider 凭据或本地路径。

默认 Prompt 要求：

```text
你是一个专业翻译助手。请将用户提供的文本翻译为 {target_language}。
源语言：{source_language}
翻译风格：{style}

要求：
1. 保留原文段落结构。
2. 保留代码、变量名、链接、产品名和专有名词。
3. 遵循术语表中命中的术语。
4. 不添加原文不存在的信息。
5. 只输出译文，不输出解释。

术语表：
{glossary}

待翻译文本：
{text}
```

### 5.5 术语表

用户可以维护个人本地术语表。

术语字段：

- 原词。
- 译词。
- 适用目标语言。
- 说明或上下文。
- 是否启用。

要求：

- 术语表保存在本地，不上传完整术语库。
- 翻译前只匹配当前源文本中出现的启用术语。
- Prompt 中只注入命中的术语，降低隐私和 token 成本。
- 支持新增、编辑、删除、启用和停用术语。
- 支持按原词或译词搜索。
- 原词不能为空，译词不能为空。
- 同一目标语言下不允许重复原词。

首版不要求导入导出，后续可按需求扩展 CSV。

### 5.6 OCR 原文编辑

截图翻译结果窗口允许用户编辑 OCR 原文后重新翻译。

要求：

- 默认展示 OCR 识别结果。
- 用户可以进入编辑状态修改原文。
- 修改后点击重新翻译，使用编辑后的文本请求 Provider。
- 历史记录保存实际发送的原文，而不是原始 OCR 文本。
- 不保存截图图片。

### 5.7 浮窗位置偏好

用户可以选择翻译窗口出现位置。

首批选项：

- 屏幕中央。
- 鼠标附近。
- 上次位置。

默认值：

- 快速翻译：屏幕中央。
- 截图翻译：优先靠近截图区域，空间不足时回退到屏幕中央。

该功能优先级低于语言、Prompt 和术语。

## 6. 数据与隐私

### 6.1 配置存储

新增配置可以保存在 `UserDefaults`：

```json
{
  "translationPreferences": {
    "sourceLanguage": "auto",
    "targetLanguage": "auto-opposite",
    "style": "accurate",
    "customPromptEnabled": false
  }
}
```

自定义 Prompt 和术语表可以保存在 `Application Support/Parrot/` 下的本地 JSON 文件。

### 6.2 隐私边界

- API Key 继续只保存在 Keychain。
- 默认不上传截图图片。
- 翻译请求只发送待翻译文本、命中的术语和当前 Prompt。
- 不把完整术语库发送给 Provider。
- 不把 Prompt、术语或历史记录写入日志。

## 7. 验收标准

### 7.1 设置快捷键

- 用户可以在 Settings 的 `Shortcuts` 中配置 `Open Settings`。
- 在其他 App 前台时，按下设置快捷键可以打开 Settings。
- 快捷键冲突或无效时，保存按钮禁用或展示错误。
- 保存后无需重启即可生效。

### 7.2 语言选择

- 快速翻译窗口可以选择源语言和目标语言。
- 截图翻译结果窗口可以选择源语言和目标语言。
- 一键互换后，重新翻译使用新的语言方向。
- 语言选择会影响实际 Prompt 和 Provider 请求。

### 7.3 Prompt 设置

- Settings 中能看到默认 Prompt。
- 用户可以启用自定义 Prompt。
- 自定义 Prompt 缺少必要变量时不能保存。
- 恢复默认后，翻译结果回到默认 Prompt 行为。

### 7.4 术语表

- 用户可以新增、编辑、删除、启用和停用术语。
- 翻译命中术语时，译文遵循术语设置。
- 未命中的术语不会注入 Prompt。
- 术语表不包含 API Key 或截图图片。

### 7.5 OCR 原文编辑

- 截图 OCR 成功后，用户可以编辑原文。
- 编辑后重新翻译使用编辑文本。
- 历史记录保存编辑后的原文和译文。

## 8. 版本拆分建议

### 8.1 优先实现

1. 设置快捷键。
2. 语言选择和一键互换。
3. 翻译风格。
4. 默认 Prompt 展示和自定义 Prompt。
5. 术语表。

### 8.2 后续实现

1. OCR 原文编辑。
2. 浮窗位置偏好。
3. 术语 CSV 导入导出。
4. 历史记录按语言和术语筛选。

## 9. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 设置项过多 | 破坏轻量体验 | 高频语言栏放在翻译窗口，低频项放在 Settings |
| 自定义 Prompt 写坏 | 翻译不可用 | 保存前校验变量，失败时回退默认 Prompt |
| 术语表增加 token 成本 | 短文本变慢 | 只注入命中的术语 |
| 多语言识别不准 | 翻译方向错误 | 保留手动语言选择和一键互换 |
| OCR 误识别 | 翻译错误 | 支持编辑 OCR 原文后重新翻译 |

## 10. 结论

V1 应围绕翻译偏好系统建设，而不是堆叠设置项。语言选择、Prompt、术语和风格都必须进入真实翻译链路，并且保留 Parrot 的轻量使用方式：按快捷键、得到译文、复制后回到原任务。
